Usage

All the endpoints in this API are protected with JWT. In order to perform CRUD operations on the notes, users need to register themselves.

The next section demonstrates the usage* for registration of new users and logging in for existing users.

*It is imperative to note that all POST/PUT requests must have a content-type of application/json

Users

Register a New User

Endpoint URL: /api/register Method: POST

To register a new user send a POST that looks as follows.

Example Request 
POST http://localhost:4000/api/register HTTP/1.1
Content-Type: application/json

{
  "first_name": "Ashwin",
  "last_name": "Sundararajan",
  "email": "a@b.com",
  "password": "password"
}
Example Output 
{
  "message": "User registered successfully",
  "token":
    "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpZCI6MywiaWF0I
    joxNTM4MDY2NzE2LCJleHAiOjE1MzgwODgzMTZ9.zHA6xi17VrLTTdzhxCr51lOpeHQleawh_IsuZz38LbE"
}

All 4 fields are required for registration. Upon successful registration, you will receive an acknowledgement and a JSON Web Token. This token is required for any request involving the notes API.

Login User

Endpoint URL: /api/login Method: POST

To sign in send a POST request that looks as follows.

Example Request 
POST http://localhost:4000/api/login HTTP/1.1
Content-Type: application/json

{
  "email": "a@b.com",
  "password": "password"
}
Example Output 
{
  "message": "Login successful",
  "token":
    "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpZCI6MywiaWF0I

Upon success, you will receive an acknowledgement and a JSON Web Token. This token is required for any request involving the notes API.

Notes

For all requests in this section the users need to authenticate themselves using the token provided when they signed up or logged in. The Bearer token must be attached to every request in the header.

For instance, a get request to the /api/notes end point will look like this.

Example Request 
GET http://localhost:4000/api/notes HTTP/1.1
Authorization: Bearer
  eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpZCI6MSwiaWF0IjoxNTM4MDQ2MTM2LCJleHAiOjE1MzgwNjc3MzZ9.
  BhNzdgd2SQ53QWIA0rK9faJ4ILEFnvuehdYfpINhxDo

All the following examples assume you have set the header appropriately.

Add a New Note

Endpoint URL: /api/notes Method: POST

Make a post request to the endpoint in order to add a new note. The post request must have content-type set to application/json and the request body must contain a title and a textBody. Optionally you can also specify tags associated with the notes. In order to add tags, include an array of strings in the request body with the key tags. If the operation was successful, it will return a message and the id of the new note.

Example Usage 
POST http://localhost:4000/api/notes HTTP/1.1
Content-Type: application/json

{
  "title": "Note 5",
  "textBody": "The quick brown fox jumps over the lazy dog",
  "tags": ["Entertainment", "Pets"]
}
Example Output 
{
  "message": "Note added successfully",
  "id": 8
}

Get Note by ID

Endpoint URL: /api/notes/:id Method: GET

Use this endpoint to get details for one note specified by ID. The note must have been authored by the currently logged in user. A get request to /api/notes/:id where :id refers the unique id of the note will yield the following results.

Example Output 
{
  "id": 5,
  "title": "Note Five",
  "textBody": "The quick brown fox jumps over the lazy dog",
  "tags": [
    "Entertainment",
    "Pets",
    "Dogs"
  ]
}

Update Note by ID

Endpoint URL: /api/notes/:id Method: PUT

Use this endpoint to update details of a note specified by ID. The note must have been authored by the currently logged in user. Make a PUT request to /api/notes/:id, where :id refers to the ID of the note you'd like to update. At least one of the fields must be specified in order to successfully update the note.

Example Usage 
PUT http://localhost:4000/api/notes/6 HTTP/1.1
Content-Type: application/json

{
  "tags": [
    "Entertainment",
    "Pets",
    "Dogs"
  ]
}
Example Output 
{
  "message": "Note with id 6 updated successfully"
}

Delete Note by ID

Endpoint URL: /api/notes/:id Method: DELETE

Use this endpoint to delete a note specified by ID. The note must have been authored by the currently logged in user. This action is irreversible. Make a DELETE request to /api/notes/:id, where :id refers to the ID of the note you'd like to delete. Upon successful deletion you will receive the following response.

Example Usage 
DELETE http://localhost:4000/api/notes/7 HTTP/1.1
Example Output 
{
  "message": "Note with id 7 has been deleted successfully"
}
