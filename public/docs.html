RESTful API Endpoints

/api/register

POST / - expects an object with keys "username" and "password", returns the ID of the registered user.

/api/login

POST / - expects an object with keys "username" and "password", returns a JWT that expires in an hour.

/api/notes

All notes endpoints expect a valid JWT sent in the "Authorization" header.

GET / - returns all notes with the same id as the user the JWT is assigned to.

POST / - expects an object with keys of "title" and "content" and strings for values. Both a title and content must be provided. Returns a successful message and the note added.

GET /:id - returns a specific note with an id matching the URL, as long as it belongs to the user and it exists.

PUT /:id - expects an object like POST, edits the note with an id matching the URL, as long as it exists and belongs to the user. Will succeed if either title or content are provided. Returns a successful message and the modified note.

DELETE /:id - deletes the note with an id matching the URL, as long as it exists and belongs to the user. Returns a successul message.
